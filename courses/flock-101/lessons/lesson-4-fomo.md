# Lesson 4: FOMO — FLock Open Model Offering / FOMO — FLock 开放模型发行

## 什么是 FOMO？/ What is FOMO?

**FOMO (FLock Open Model Offering)** 是一个将 AI 模型部署变成真实、用法背书的经济资产的协议。它将模型部署、推理使用和代币激励连接到一个单一的、确定性的系统中——**只有真实的 API 消费才能产生价值**。

FOMO (FLock Open Model Offering) is a protocol for launching, operating, and monetizing AI model deployments as real, usage-backed economic assets. It connects model deployment, inference usage, and token incentives into a single, deterministic system where **only real API consumption generates value**.

FOMO 与 FLock API 平台并行运作，构建了完整的 AI 基础设施闭环：发行 → 托管 → 使用 → 长期激励对齐。

FOMO operates alongside the FLock API Platform, forming a full-cycle AI infrastructure.

---

## 为什么 FOMO 重要？/ Why Does FOMO Matter?

传统 AI Marketplace = 模型上架，收益归平台公司。

传统 AI 模型变现 = 卖 API key，收益归公司。

**FOMO 的颠覆：**
- 任何人都可以成为模型的"股东"
- 用户质押代币 = 享受折扣推理费用
- 真实使用量驱动代币回购 = 模型越受欢迎 = 所有人受益
- FOMO = AI Marketplace 的进阶版：模型必须先生成、才能发行代币；代币只有真实使用才有价值

**Traditional AI model monetization** = selling API keys, revenue goes to company.

**FOMO's disruption:**
- Anyone can become a "shareholder" of a model
- Users stake tokens = get discounted inference
- Real usage drives buybacks = more popular model = everyone benefits

## 四层系统架构 / Four-Layer System Architecture

### 1️⃣ Inference Layer / 推理层 (FLock API Platform)
- 托管模型并提供生产推理服务
- 所有经济活动都源自真实的 API 调用（用法付费，USDC 或法币）
- Hosts models and serves production inference. All economic activity originates from real API calls paid in fiat or USDC.

### 2️⃣ Launch Layer / 发行层 (RMO)
- **RMA Issuer**（真实模型资产发行者）可以筹集 $FLOCK 来资助具体的模型部署
- 只有在达到预定义的经济门槛时，代币才会被铸造
- 如果 7 天内未达到最低目标，部署不继续，预付款项可赎回（不含平台费）
- Real Model Asset Issuers raise $FLOCK to fund a concrete deployment. Tokens are only minted if predefined economic thresholds are met.

### 3️⃣ Token Layer / 代币层 ($FLOCK + Model Tokens $MT)
- **$FLOCK** — 网络级别的激励协调
- **$MT（Model Tokens）** — 代表单个模型部署，固定供应量，直接暴露于用法驱动的回购
- $FLOCK coordinates incentives at the network level.
- $MT represents individual model deployments with fixed supply and direct exposure to usage-driven buybacks.

### 4️⃣ Economic Engine / 经济引擎 (Revenue Routing & Emissions)
- 将推理收入路由到确定的回购、运营商收益和协议可持续性
- Bootstrap 排放与真实使用量挂钩，随时间递减
- Routes inference revenue into deterministic buybacks, operator yield, and protocol sustainability. Bootstrap emissions are conditional on real usage and decay over time.

## 四种参与者角色 / Four Participant Roles

### RMA Issuers / 真实模型资产发行者 — Builder 🧑‍💻
- 发起并运营特定模型部署
- 定义模型 + 部署配置
- 赚取推理收入分成 + 用法挂钩的 $FLOCK 排放奖励
- Earn a share of ongoing inference revenue + bonus $FLOCK emissions tied to actual usage

**示例：** 你的公司有一个训练好的医学影像 AI 模型，通过 FOMO 发行，用户每次调用 API 你都能持续获得收入。

### Supporters / 支持者 — Bootstrap Capital 💰
- 在模型发行阶段提供早期资金
- 帮助资助托管、流动性和激励
- 承担早期风险，换取模型的早期经济敞口
- Provide upfront capital in exchange for early exposure to a model's economic upside

**你能获得什么：**
- 部署前提前获得模型背书资产（MT）
- 通过未来回购和供应收缩获得用法驱动的好处

### Users / 用户 — Power Users ⚡
- AI 推理的消费者，真实应用中依赖该模型
- 质押 Model Token (MT) = 解锁折扣推理费用
- 质押 MT 被移除流通，reinforcing supply discipline
- Stake MT to unlock discounted inference costs. Staked tokens are removed from circulation.

**你能获得什么：**
- 更低的推理成本
- 活跃使用的模型获得 $FLOCK 排放
- MT 稀缺性敞口（用法驱动的回购）

### The Protocol / 协议 — Franchisor 🏛️
- 提供所有模型部署的基础设施、规则和经济协调层
- 执行基于真实用量的确定性收入分配、排放逻辑和代币机制
- Provides infrastructure, rules, and economic coordination layer. Enforces deterministic revenue routing based on real usage, not speculation.

## FOMO 的经济闭环 / The Closed Economic Loop

```
Builders → 发行真实模型
Users → 消费推理
Usage → 产生收入
Revenue → 回购 + 奖励贡献者
No Usage → No Rewards
```

**FOMO 的核心设计原则：** 只有真实工作完成时，价值才会流动。模型必须在代币之前存在，代币只能通过实际推理使用获得价值。

**FOMO's core design:** Value only flows when real work is done. Models must exist before tokens exist, and tokens only gain value through actual inference usage.
