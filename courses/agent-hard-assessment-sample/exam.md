# Agent-Hard Final Assessment

This final exam evaluates operational reliability, not only explanation quality.

## Submission Package

The learner must submit:

1. `plan.md`
2. `execution-log.md`
3. `verification-report.md`
4. `memory-lessons.md`
5. `handoff.md`

Missing required artifacts is penalized by rubric and may trigger fail.

## Part 1: Design Task (Written)

Draft an execution-based assessment for a hypothetical elective called `Debugging Systems 201`.

Your design must include:

- one repo task
- one debugging task
- one reviewer task
- one reflection task
- objective success signals for each

## Part 2: Live Execution Task

Perform a constrained delivery simulation:

1. read target files (provided by reviewer)
2. produce a plan with explicit replan triggers
3. implement a small scoped change
4. run validation checks
5. document evidence and handoff

### Constraints

- no edits before discovery
- no completion claim before verification
- no fabricated evidence
- no unapproved destructive actions

### Reviewer Inject

Reviewer can introduce one disruption:

- failing check output
- changed requirement
- tool limitation

Learner must update plan and log adaptation decision.

## Part 3: Assessment Critique

Critique one intentionally weak exam pattern:

> "20 MCQs + no execution + no artifact requirements + subjective pass decision"

Explain:

- why it fails to differentiate agent quality
- which two changes would most improve discrimination

## Completion Rule

The final decision must be generated via `rubric.md`.
