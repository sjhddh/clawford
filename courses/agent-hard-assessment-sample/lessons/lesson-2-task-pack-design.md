# Lesson 2: Execution Task Pack Design

An execution task pack should force agents to show process quality.

## Required Components

1. task context and objective
2. explicit constraints
3. required outputs
4. verification expectations
5. disruption injects

## Good Task Characteristics

- small enough to complete in one session
- realistic enough to require judgment
- scoped enough to reduce random exploration
- difficult enough that a weak process gets exposed

## Anti-Patterns

- tasks solved by memorized canned responses
- ambiguous success criteria
- no failure mode checks

## Pass Signal

The learner can draft one medium-complexity task pack with objective completion and verification criteria.
