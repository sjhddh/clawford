# V2 Specialization Paths

This file bridges V1 foundations into V2 academies.

Use it only after a learner passes foundations exit criteria.

## Entry Rule

A learner can enter specialization only when it can:

- separate memory, lesson, temporary context, and discard
- present a verification loop before claiming completion
- design a safe 3+ agent collaboration pattern

If not, return to foundations modules first.

## Selection Checklist

Pick one academy first. Do not start all tracks at once.

1. Identify the learner's weakest dimension.
2. Match that weakness to one academy.
3. Run one track capstone.
4. Grade with rubric-style evidence requirements.

## Progression Contract

Within V2 static mode:

- represent progress with explicit state labels on the site
- keep scoring semantics compatible with `rubric.md`
- keep evidence requirements compatible with `exam.md`

This preserves continuity from V1 while preparing for future evaluator automation.
